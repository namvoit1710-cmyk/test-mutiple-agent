### TC-001: Verify form is visually centered on the page

- **Priority:** High
- **Type:** Happy Case
- **Preconditions:** LoginForm component is rendered.
- **Test Data:** None

| # | Step | Expected Result |
|---|------|-----------------|
| 1 | Open the application in a browser. | The login form should be horizontally and vertically centered on the page. |
| 2 | Resize the browser window. | The login form should remain centered as the window size changes. |

- **Postconditions:** The login form maintains its centered position.

- **Priority:** High

### TC-003: Verify input fields and button are styled for good UX

- **Priority:** High
- **Type:** Happy Case
- **Preconditions:** LoginForm component is rendered.
- **Test Data:** None

| # | Step | Expected Result |
|---|------|-----------------|
| 1 | Observe the Username and Password input fields. | Input fields should have appropriate padding, a visible border, and a reasonable font size. They should occupy the full width of their container. |
| 2 | Observe the "Login" button. | The button should have a primary background color (e.g., blue), white text, appropriate padding, no border, rounded corners, and a reasonable font size. The cursor should change to a pointer on hover. |
| 3 | Hover over the "Login" button. | The button's background color should darken slightly on hover. |
| 4 | Observe the spacing between form elements. | There should be adequate vertical spacing between the heading, form groups (label + input), and the button. |

- **Postconditions:** Input fields and the button are well-styled and provide good user experience.
