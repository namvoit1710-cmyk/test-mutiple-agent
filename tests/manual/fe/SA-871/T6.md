# Manual Test Cases: SA-871 / T6 [FE]

Source spec: `app/docs/spec/SA-871.md`
Source plan task: `T6` in `app/docs/plan/SA-871.md`

### TC-001: Verify "Submit" button is disabled when "Full Name" field is empty

- **Priority:** Critical
- **Type:** Negative Case
- **Preconditions:** The form page is loaded. The "Full Name" input field is empty.
- **Test Data:** Empty string

| # | Step | Expected Result |
|---|---|---|
| 1 | Navigate to the form page. | The "Full Name" input field is visible and empty. |
| 2 | Observe the "Submit" button. | The "Submit" button is disabled. |
| 3 | Click outside the input field (trigger onBlur). | The "Submit" button remains disabled. |

- **Postconditions:** "Submit" button is disabled.

### TC-002: Verify "Submit" button is enabled when "Full Name" field is valid

- **Priority:** Critical
- **Type:** Happy Case
- **Preconditions:** The form page is loaded. The "Full Name" input field is ready for input.
- **Test Data:** "Valid Name"

| # | Step | Expected Result |
|---|---|---|
| 1 | Navigate to the form page. | The "Full Name" input field is visible. |
| 2 | Enter "Valid Name" into the input field. | The text "Valid Name" is displayed. |
| 3 | Click outside the input field (trigger onBlur). | No error message is displayed. The input field border changes to a normal/success state. |
| 4 | Observe the "Submit" button. | The "Submit" button is enabled. |

- **Postconditions:** "Submit" button is enabled.

### TC-004: Verify "Submit" button is disabled with invalid characters

- **Priority:** Critical
- **Type:** Negative Case

### TC-005: Verify "Submit" button is disabled with maximum length violation

- **Priority:** High
- **Type:** Negative Case
- **Preconditions:** The form page is loaded. The "Full Name" input field is ready for input.
- **Test Data:** A string of 51 alphabetic characters and spaces.

| # | Step | Expected Result |
|---|---|---|
| 1 | Navigate to the form page. | The "Full Name" input field is visible. |
| 2 | Enter a 51-character string into the input field. | The input field should prevent entry of the 51st character, allowing only 50 characters. |
| 3 | Click outside the input field (trigger onBlur). | An error message indicating the maximum length has been exceeded should appear (or the existing error message for invalid characters if numbers/specials were also included). |
| 4 | Observe the "Submit" button. | The "Submit" button is disabled. |

- **Postconditions:** "Submit" button is disabled.
