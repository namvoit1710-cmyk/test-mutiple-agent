# Manual Test Cases: SA-871 / T2 [FE]

Source spec: `app/docs/spec/SA-871.md`
Source plan task: `T2` in `app/docs/plan/SA-871.md`

### TC-001: Verify input field state when empty

- **Priority:** Critical
- **Type:** Negative Case
- **Preconditions:** The form page is loaded. The "Full Name" input field is empty.
- **Test Data:** Empty string

| # | Step | Expected Result |
|---|---|---|
| 1 | Navigate to the form page. | The "Full Name" input field is visible and empty. |
| 2 | Click outside the input field (trigger onBlur). | An error message "Please enter your full name." appears below the input field. The input field border remains in its default state (or indicates an error).
| 3 | Observe the "Submit" button. | The "Submit" button is disabled. |

- **Postconditions:** Error message is displayed. Submit button is disabled.

### TC-002: Verify input field state with valid data

- **Priority:** Critical
- **Type:** Happy Case
- **Preconditions:** The form page is loaded. The "Full Name" input field is ready for input.
- **Test Data:** "Jane Smith"

| # | Step | Expected Result |
|---|---|---|
| 1 | Navigate to the form page. | The "Full Name" input field is visible. |
| 2 | Enter "Jane Smith" into the input field. | The text "Jane Smith" is displayed. |
| 3 | Click outside the input field (trigger onBlur). | No error message is displayed. The input field border changes to a normal/success state. |
| 4 | Observe the "Submit" button. | The "Submit" button is enabled. |

- **Postconditions:** No error message. Submit button is enabled.

### TC-005: Verify input field state with maximum length violation

- **Priority:** High
- **Type:** Negative Case
- **Preconditions:** The form page is loaded. The "Full Name" input field is ready for input.
- **Test Data:** A string of 51 alphabetic characters and spaces.

| # | Step | Expected Result |
|---|---|---|
| 1 | Navigate to the form page. | The "Full Name" input field is visible. |
| 2 | Enter a 51-character string into the input field. | The input field should truncate or prevent entry of the 51st character, allowing only 50 characters. |
| 3 | Click outside the input field (trigger onBlur). | An error message indicating the maximum length has been exceeded should appear (or the existing error message for invalid characters if numbers/specials were also included). The input field border indicates an error. |
| 4 | Observe the "Submit" button. | The "Submit" button is disabled. |

- **Postconditions:** Error message is displayed. Submit button is disabled.
