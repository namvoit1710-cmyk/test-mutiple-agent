# Manual Test Cases: SA-848 [BE]

Source spec: `app/docs/spec/SA-848.md`

### TC-001: Successful authentication

- **Priority:** Critical
- **Type:** Happy Case
- **Endpoint:** `POST /api/auth/login`
- **Preconditions:** User `testuser` with password `password123` exists in the database.
- **Request:**
  - Headers: `Content-Type: application/json`
  - Body:
    ```json
    { "username": "testuser", "password": "password123" }
    ```
- **Expected Response:**
  - Status: `200 OK`
  - Body:
    ```json
    { "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "user": { "username": "testuser" } }
    ```
- **Postconditions:** A valid session/token is generated for the user.

### TC-002: Authentication with invalid password

- **Priority:** High
- **Type:** Negative Case
- **Endpoint:** `POST /api/auth/login`
- **Preconditions:** User `testuser` with password `password123` exists in the database.
- **Request:**
  - Headers: `Content-Type: application/json`
  - Body:
    ```json
    { "username": "testuser", "password": "wrongpassword" }
    ```
- **Expected Response:**
  - Status: `401 Unauthorized`
  - Body:
    ```json
    { "error": "Invalid username or password" }
    ```
- **Postconditions:** No token is generated.
