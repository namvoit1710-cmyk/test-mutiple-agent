# Manual Test Cases: SA-875 [FE]

Source spec: `app/docs/spec/SA-875.md`

### TC-001: Redirect unauthenticated user from Home to Login

- **Priority:** Critical
- **Type:** Negative Case
- **Preconditions:** User is not logged in (no auth token exists).
- **Test Data:** Browser in incognito mode or existing session cleared.

| # | Step | Expected Result |
|---|---|---|
| 1 | Open the application in a browser. | The application should load. |
| 2 | Manually navigate to the `/home` URL (or `/`). | The user should be automatically redirected to the `/login` page. |

- **Postconditions:** User is on the `/login` page.

### TC-002: Redirect authenticated user from Login to Home

- **Priority:** Critical
- **Type:** Negative Case
- **Preconditions:** User is logged in (auth token exists).
- **Test Data:** A valid auth token is present in the browser's storage (e.g., localStorage).

| # | Step | Expected Result |
|---|---|---|
| 1 | Ensure the user is logged in and has a valid token. | User is on the `/home` page or a page accessible after login. |
| 2 | Manually navigate to the `/login` URL. | The user should be automatically redirected to the `/home` page. |

- **Postconditions:** User is on the `/home` page.

### TC-003: Allow unauthenticated user to access Login page

- **Priority:** High
- **Type:** Happy Case
- **Preconditions:** User is not logged in (no auth token exists).
- **Test Data:** Browser in incognito mode or existing session cleared.

| # | Step | Expected Result |
|---|---|---|
| 1 | Open the application in a browser. | The application should load. |
| 2 | Manually navigate to the `/login` URL. | The user should remain on the `/login` page and the login form should be visible. |

- **Postconditions:** User is on the `/login` page.

### TC-004: Allow authenticated user to access Home page

- **Priority:** High
- **Type:** Happy Case
- **Preconditions:** User is logged in (auth token exists).
- **Test Data:** A valid auth token is present in the browser's storage (e.g., localStorage).

| # | Step | Expected Result |
|---|---|---|
| 1 | Ensure the user is logged in and has a valid token. | User is on the `/home` page or a page accessible after login. |
| 2 | Manually navigate to the `/home` URL (or `/`). | The user should remain on the `/home` page and the content of the home page should be visible. |

- **Postconditions:** User is on the `/home` page.

### TC-005: Verify token existence check for Home page guard

- **Priority:** Medium
- **Type:** Edge Case
- **Preconditions:** User is not logged in.
- **Test Data:** Attempt to access `/home`.

| # | Step | Expected Result |
|---|---|---|
| 1 | Attempt to access the `/home` page. | The application should check for the existence of an authentication token using `getToken()`. |
| 2 | Verify that if `getToken()` returns a falsy value (e.g., null, undefined, empty string), the redirect to `/login` occurs. | The user is redirected to `/login`. |

- **Postconditions:** User is on the `/login` page.

### TC-006: Verify token existence check for Login page guard

- **Priority:** Medium
- **Type:** Edge Case
- **Preconditions:** User is logged in.
- **Test Data:** Attempt to access `/login`.

| # | Step | Expected Result |
|---|---|---|
| 1 | Attempt to access the `/login` page. | The application should check for the existence of an authentication token using `getToken()`. |
| 2 | Verify that if `getToken()` returns a truthy value (e.g., a non-empty string), the redirect to `/home` occurs. | The user is redirected to `/home`. |

- **Postconditions:** User is on the `/home` page.
