# Manual Test Cases: SA-875 [FE]

Source spec: `app/docs/spec/SA-875.md`

### TC-001: Unauthenticated user attempts to access Home page

- **Priority:** Critical
- **Type:** Negative Case
- **Preconditions:** User is not logged in (no auth token).
- **Test Data:** Incognito/private browsing window.

| # | Step | Expected Result |
|---|---|-----------------|
| 1 | Open a new incognito/private browser window. |  |
| 2 | Navigate directly to the `/home` URL. | The application redirects the user to the `/login` page. |

- **Postconditions:** User is on the `/login` page.

### TC-002: Authenticated user attempts to access Login page

- **Priority:** Critical
- **Type:** Negative Case
- **Preconditions:** User is logged in (auth token exists).
- **Test Data:** Valid auth token.

| # | Step | Expected Result |
|---|---|-----------------|
| 1 | Ensure the user is logged in (e.g., by completing a successful login). |  |
| 2 | Navigate directly to the `/login` URL. | The application redirects the user to the `/home` page. |

- **Postconditions:** User is on the `/home` page.

### TC-005: User logs out and attempts to access Home page

- **Priority:** Medium
- **Type:** Edge Case
- **Preconditions:** User is logged in.
- **Test Data:** Valid auth token.

| # | Step | Expected Result |
|---|---|-----------------|
| 1 | Log in to the application. | User is on the `/home` page and logged in. |
| 2 | Navigate to the `/home` page. | User remains on the `/home` page. |
| 3 | Perform the logout action. | The user is redirected to the `/login` page after logging out. |

- **Postconditions:** User is on the `/login` page and logged out.

### TC-006: User logs in from Login page

- **Priority:** Critical
- **Type:** Happy Case
- **Preconditions:** User is not logged in.
- **Test Data:** Valid username and password.

| # | Step | Expected Result |
|---|---|-----------------|
| 1 | Navigate to the `/login` page. | The login page is displayed. |
| 2 | Enter valid username and password. | Input fields accept data. |
| 3 | Click the login button. | The user is redirected to the `/home` page after successful login. |

- **Postconditions:** User is on the `/home` page and logged in.
