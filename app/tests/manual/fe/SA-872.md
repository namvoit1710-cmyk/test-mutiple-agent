
# Manual Test Cases: SA-872 [FE]

Source spec: `app/docs/spec/SA-872.md`

### TC-001: Verify Home Page Content and Basic Centering

- **Priority:** Critical
- **Type:** Happy Case
- **Preconditions:** User has navigated to the home page URL.
- **Test Data:** Default application data.

| # | Step | Expected Result |
|---|------|-----------------|
| 1 | Observe the home page. | The main heading "Welcome" (or app name), a descriptive subheading, and a "Get Started" button are visible. |
| 2 | Verify the content block is centered. | The heading, subheading, and button are visually centered both horizontally and vertically within the viewport. |
| 3 | Check the main container height. | The page content fills the entire viewport height without any extra vertical space or scrollbars (unless content exceeds viewport). |

- **Postconditions:** User is on the home page, content is displayed as expected.

### TC-002: Verify Horizontal and Vertical Centering Across Viewports

- **Priority:** Critical
- **Type:** Happy Case
- **Preconditions:** User has navigated to the home page URL.
- **Test Data:** Browser developer tools to resize viewport.

| # | Step | Expected Result |
|---|------|-----------------| |
| 1 | Resize the browser window to a wide desktop width. | The content block remains perfectly centered horizontally and vertically. |
| 2 | Resize the browser window to a tablet width (e.g., 768px). | The content block remains perfectly centered horizontally and vertically. |
| 3 | Resize the browser window to a mobile width (e.g., 375px). | The content block remains perfectly centered horizontally and vertically. |

- **Postconditions:** User is on the home page, centering is maintained across different screen sizes.

### TC-003: Verify Responsive Layout and Scaling

- **Priority:** High
- **Type:** Happy Case
- **Preconditions:** User has navigated to the home page URL.
- **Test Data:** Browser developer tools to simulate various devices.

| # | Step | Expected Result |
|---|------|-----------------|
| 1 | View the page on a mobile device simulation. | Text size, button size, and spacing scale down appropriately for the smaller viewport while maintaining centered alignment. No horizontal scrollbars appear due to content overflow. |
| 2 | Gradually increase viewport width from mobile to desktop. | The layout scales smoothly, text and spacing adjust gracefully, and the centered alignment is preserved throughout the transition. |

- **Postconditions:** User is on the home page, layout is responsive and scales correctly.

### TC-010: Verify No Unnecessary Scrollbars

- **Priority:** High
- **Type:** Negative Case
- **Preconditions:** User has navigated to the home page URL.
- **Test Data:** Visual inspection across various viewport sizes.

| # | Step | Expected Result |
|---|------|-----------------|
| 1 | Observe the page in a standard desktop viewport. | No horizontal or vertical scrollbars are present on the main `body` element, indicating the content fits perfectly within the viewport. |
| 2 | Resize the viewport to various sizes, including small mobile sizes. | No unnecessary horizontal scrollbars appear. Vertical scrollbars only appear if the content legitimately exceeds the viewport height (which should not be the default case for this page). |

- **Postconditions:** User is on the home page, no unintended scrollbars are visible.
