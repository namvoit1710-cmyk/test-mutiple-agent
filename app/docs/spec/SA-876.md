**Scope:** A reusable password input component with show/hide toggle, uncontrolled by default with ref API.

## Overview

A single `<PasswordControl />` component that renders a password `<input>` with an inline visibility toggle button. The toggle switches the input `type` between `"password"` and `"text"`. No strength meter, no confirmation field, no validation rules.

## Behavior

### Toggle

- Default state: password hidden (input `type="password"`)
- Clicking the toggle button flips `type` between `"password"` and `"text"`
- Toggle button is `type="button"` to prevent form submission
- Toggle button has dynamic `aria-label`: `"Show password"` / `"Hide password"`

### Icons

- Eye and eye-off icons are inline SVGs embedded in the component (no icon library dependency)

### Accessibility

- Toggle button: `aria-label` reflects current state, `tabindex="0"` for keyboard access
- Input label: not provided by the component â€” parent is responsible for labeling via `id` + `<label htmlFor>`

## Out of Scope

- Strength meter or password rules
- Confirmation / repeat-password field
- Validation feedback or error display
- Controlled component mode (`value` + `onChange` props)
- Label rendering (parent provides `<label>` externally)
