# SA-873: Login page ui

## Description

h1. FE Task: Implement Login Page

h2. 1. Overview

Create a minimal and responsive login page. The main feature is a centralized "Form Card" containing inputs for email and password, followed by a full-width submit button at the bottom of the card.

h2. 2. Layout & Structure

* *Main Container:* A full-screen wrapper (e.g., {{min-height: 100vh}}) that perfectly centers the Form Card both vertically and horizontally.
* *Form Card:* A distinct container (card) holding the form elements. It should have clear boundaries (e.g., background color, padding, border-radius, and a subtle drop shadow) to stand out from the page background.
* *Form Layout:* Elements inside the card should be stacked vertically (column layout) with consistent spacing between them.

h2. 3. Component Details

* *Email Field:*
** *Type:* {{email}}
** *Label/Placeholder:* "Email Address"
** *Required:* Yes
* *Password Field:*
** *Type:* {{password}}
** *Label/Placeholder:* "Password"
** *Required:* Yes
* *Submit Button:*
** *Type:* {{submit}}
** *Text:* "Login" or "Sign In"
** *Width:* 100% (Full width of its parent container)
** *Placement:* Positioned directly below the password input.

h2. 4. CSS & Styling Guidelines

* *Centering:* Use CSS Flexbox ({{display: flex; justify-content: center; align-items: center;}}) or CSS Grid on the main container to center the card.
* *Button Sizing:* Ensure the submit button uses {{width: 100%;}} or {{display: block; width: 100%;}} so it spans the entire horizontal space inside the form card.
* *Spacing:* Apply appropriate {{padding}} inside the card and {{margin-bottom}} (or {{gap}}) between the input fields and the button for a clean look.

h2. 5. Acceptance Criteria

* [ ] The form card is perfectly centered on the screen regardless of the viewport size.
* [ ] The email input correctly uses the {{type="email"}} attribute to trigger native browser validation and mobile keyboards.
* [ ] The password input securely masks the user's typing.
* [ ] The submit button expands to take up the full width of the form card.
* [ ] The layout is responsive, ensuring the card does not touch the very edges of the screen on small mobile devices (e.g., retains some outer margin/padding).
