# SA-875: Handle authen for app

## Description

h1. FE Task: Implement Authentication Routing Guards

h2. 1. Overview

Implement route protection to ensure users can only access appropriate pages based on their authentication status. We need to secure the Home page from unauthenticated guests, and prevent logged-in users from returning to the Login page.

h2. 2. Technical Requirements

* *Dependency:* Utilize the {{getToken()}} function from our {{auth}} utility (e.g., {{localStorage}}) to verify if the user is currently authenticated.
* *Router Integration:* Apply this logic at the router level (e.g., using React Router guards, Vue Router {{beforeEach}}, or Next.js Middleware, depending on the framework).

h2. 3. Routing Logic Specifications

* *Private Route Guard (Target:* {{/home}} or {{/}}):
** _Condition:_ A user attempts to access the home page.
** _Check:_ Validate if the auth token exists.
** _Action:_ If *NO* token is found, immediately redirect the user to {{/login}}.
* *Auth Route Guard (Target:* {{/login}}):
** _Condition:_ A user attempts to access the login page.
** _Check:_ Validate if the auth token exists.
** _Action:_ If a token *IS* found, immediately redirect the user to {{/home}}.

h2. 4. Acceptance Criteria

* [ ] *Test Case 1:* Open an incognito window (no token). Manually type the {{/home}} URL. The app must automatically redirect to {{/login}}.
* [ ] *Test Case 2:* Log in to the application (token is saved). Manually type the {{/login}} URL. The app must automatically redirect to {{/home}}.
* [ ] *Test Case 3:* An unauthenticated user can successfully view and stay on the {{/login}} page.
* [ ] *Test Case 4:* An authenticated user can successfully view and stay on the {{/home}} page.
