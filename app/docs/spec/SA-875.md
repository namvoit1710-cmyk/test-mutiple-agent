# SA-875: Handle authen for app

## Description

{noformat}# Auth App — Design Spec

**Date:** 2025-06-13
**Supersedes:** `2025-06-13-auth-routing-guards-design.md`, `2025-06-13-password-control-design.md`
**Scope:** Route protection (React Context + React Router), login form with email + password, homepage with welcome greeting, password input component with show/hide toggle.

## Overview

A two-page React app: **Login** (centered form card with email + password) and **Home** (centered welcome greeting + slogan). Route guards prevent unauthenticated access to `/home` and authenticated access to `/login`. Auth state is in-memory only — no backend, no persistence.

## Architecture

```
AuthProvider (Context)
  └── BrowserRouter
        ├── /login  → AuthRoute guard  → LoginPage
        └── /home   → PrivateRoute guard → HomePage
```

**Auth flow:** Login form validates email + password → `login(email, password)` sets user in context → guard redirects to `/home`.

## Auth Context API

```ts
interface AuthUser {
  email: string;
  password: string;
}

interface AuthContextValue {
  isAuthenticated: boolean;
  user: AuthUser | null;
  login: (email: string, password: string) => void;
  logout: () => void;
}
```

- `login(email, password)` — sets `user` and `isAuthenticated = true`
- `logout()` — clears `user`, sets `isAuthenticated = false`
- State is in-memory only — no `localStorage`, no backend call

## Routing Guards

### PrivateRoute (`/home`)

| Condition | Behavior |
|---|---|
| `!isAuthenticated` | `<Navigate to="/login" replace>` |
| `isAuthenticated` | Render `children` |

### AuthRoute (`/login`)

| Condition | Behavior |
|---|---|
| `isAuthenticated` | `<Navigate to="/home" replace>` |
| `!isAuthenticated` | Render `children` |

## Password Control

### Overview
A password `<input>` with an inline show/hide toggle button. The toggle switches the input `type` between `"password"` and `"text"`.

**Deviation from original spec:** Controlled mode replaces the original ref API. No `forwardRef`, no `useImperativeHandle`.

### Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `value` | `string` | `""` | Current input value (controlled) |
| `onChange` | `(value: string) => void` | — | Called on input change with the new string value |
| `placeholder` | `string` | `""` | Placeholder text |
| `disabled` | `boolean` | `false` | Disables input and toggle button |
| `className` | `string` | `""` | Additional CSS class on wrapper |
| `id` | `string` | — | `id` attribute on the `<input>` |
| `name` | `string` | — | `name` attribute on the `<input>` |
| `autoComplete` | `string` | `"current-password"` | `autocomplete` attribute |

### Behavior

| Trigger | Result |
|---|---|
| Initial state | Input `type="password"`, toggle shows eye icon |
| Click toggle | Flips `type` between `"password"` and `"text"`, icon toggles eye / eye-off |
| Toggle disabled | Both input and button get `disabled` attribute |
| Toggle `type` | `type="button"` — prevents form submission |

### Icons
Inline SVGs embedded in the component:
- **Eye (show):** An open eye SVG path
- **Eye-off (hide):** An eye with a slash SVG path

### Accessibility
- Toggle `aria-label`: `"Show password"` / `"Hide password"`
- Input: parent provides `<label htmlFor>` externally

### CSS Classes (BEM)

| Class | Element | Key Styles |
|---|---|---|
| `.password-control` | Wrapper div | `display: flex`, inline flex container |
| `.password-control__input` | `<input>` | `flex: 1`, no border on focus |
| `.password-control__toggle` | `<button>` | Square aspect ratio, centered icon, `cursor: pointer` |

## Acceptance Criteria

| # | Scenario | Expected |
|---|---|---|
| 1 | No auth → navigate to `/home` | Redirected to `/login` |
| 2 | Authenticated → navigate to `/login` | Redirected to `/home` |
| 3 | No auth → navigate to `/login` | Login form renders |
| 4 | Authenticated → navigate to `/home` | Welcome greeting shows email, logout visible |
| 5 | Submit login with valid email + password | Redirected to `/home`, greeting shows email |
| 6 | Submit login with empty email | Validation error shown |
| 7 | Submit login with empty password | Validation error shown |
| 8 | Submit login with invalid email format | Validation error shown |
| 9 | Click logout on home | Redirected to `/login` |
| 10 | Click password toggle | Input type flips, icon flips |
| 11 | PasswordControl disabled | Both input and toggle are disabled |

## Out of Scope

- Backend auth integration
- Persisted auth (localStorage)
- Strength meter or password rules
- Confirmation / repeat-password field
- Label rendering (parent provides externally)
- E2E tests (Playwright config exists but no test files requested)
- Responsive design beyond centered layout
