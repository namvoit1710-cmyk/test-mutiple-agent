# SA-848: Test agent

## Description

h1. Login Form Spec

h2. Goal

Create a simple login form UI component in the React frontend with username and password fields. When submitted, the form should display an alert showing the values entered by the user.

h2. User Story

As a user, I want to see a login form with username and password fields, so that I can enter my credentials and see them displayed in an alert when I click submit.

h2. Requirements

h3. Functional Requirements

# *Form Fields*
#* Username input field (text input)
#* Password input field (password input - characters should be hidden)
#* Submit button
# *Behavior*
#* When the user types in the username field, the component should track the value
#* When the user types in the password field, the component should track the value
#* When the user clicks the submit button:
#** Prevent the default form submission (no page reload)
#** Display an alert with the format: {{Username: [username_value], Password: [password_value]}}
# *No Validation*
#* No client-side validation required
#* Empty fields are allowed
#* No error messages needed

h3. Technical Requirements

# *Component Architecture*
#* Create a new {{LoginForm}} component in {{src/components/LoginForm.tsx}}
#* The component should be a functional React component using TypeScript
#* Use React hooks ({{useState}}) to manage form state
# *Styling*
#* Create a separate CSS file {{src/components/LoginForm.css}}
#* Center the form on the page
#* Style the form as a card with padding, border, and shadow for a modern look
#* Stack form elements vertically
#* Style inputs and button for good UX
# *Integration*
#* Import and render {{<LoginForm />}} in {{App.tsx}}
#* Replace the existing "Hello World" text

h2. Implementation Details

h3. LoginForm.tsx

*What to implement:*

# *Import statements:*
#* Import {{useState}} from React
#* Import the CSS file
# *State management:*
#* Create state for {{username}} (initial value: empty string)
#* Create state for {{password}} (initial value: empty string)
# *Event handlers:*
#* {{handleUsernameChange}}: Updates username state when user types
#* {{handlePasswordChange}}: Updates password state when user types
#* {{handleSubmit}}: 
#** Prevents default form behavior
#** Shows alert with username and password values
# *JSX structure:*
{noformat}<div className="login-container">
  <form className="login-form" onSubmit={handleSubmit}>
    <h2>Login</h2>
    
    <div className="form-group">
      <label htmlFor="username">Username</label>
      <input 
        type="text"
        id="username"
        value={username}
        onChange={handleUsernameChange}
      />
    </div>
    
    <div className="form-group">
      <label htmlFor="password">Password</label>
      <input 
        type="password"
        id="password"
        value={password}
        onChange={handlePasswordChange}
      />
    </div>
    
    <button type="submit">Login</button>
  </form>
</div>{noformat}

h2. Notes for Fresher Developers

h3. React Concepts Used

# *Functional Components*: Modern React uses functions instead of classes
# *useState Hook*: Lets you add state to functional components
#* Syntax: {{const [value, setValue] = useState(initialValue)}}
#* Returns current value and a function to update it
# *Controlled Components*: Input values are controlled by React state
# *Event Handlers*: Functions that run when user interacts (onChange, onSubmit)
# *Props*: Not used in this component, but good to know for future

h3. TypeScript Concepts

# *Type Annotations*: TypeScript adds type checking to JavaScript
# *React.FormEvent*: Type for form submit events
# *React.ChangeEvent*: Type for input change events

h3. CSS Concepts

# *Flexbox*: Used to center content and create layouts
# *Box Model*: padding, margin, border work together
# *Pseudo-classes*: {{:hover}} for interactive states
# *Class Selectors*: {{.classname}} targets elements with that class

h3. Common Mistakes to Avoid

# *Forgetting to prevent default*: Always call {{e.preventDefault()}} in form submit handler
# *Not binding state to inputs*: Use {{value={username}}} to make inputs controlled
# *Forgetting onChange handler*: Without it, inputs won't update when you type
# *Wrong event types*: Use {{React.FormEvent}} for forms, {{React.ChangeEvent<HTMLInputElement>}} for inputs
# *CSS not imported*: Must import the CSS file in the component, not just create it
# *Creating directory wrong*: Make sure to create {{components}} folder in {{src/}}
