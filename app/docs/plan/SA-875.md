# SA-875: Handle authen for app - Implementation Plan

## Overview

This plan outlines the steps to implement authentication routing guards using React Context and React Router v6, as described in the SA-875 specification. The authentication state will be in-memory only, without persistence or backend integration.

## Dependencies

- `react-router-dom` v6

## Implementation Plan

### Task 1: Install react-router-dom [FE][simple]

Install the `react-router-dom` package, which is required for routing and navigation.

### Task 2: Create Auth Context [FE][medium]

Create `src/lib/auth-context.tsx` to manage authentication state.
- Define `AuthContext` and `AuthProvider`.
- Implement `useAuth` hook to provide `isAuthenticated`, `login()`, and `logout()` functions.
- `isAuthenticated` will be a boolean state.
- `login()` will set `isAuthenticated` to `true`.
- `logout()` will set `isAuthenticated` to `false`.

### Task 3: Create PrivateRoute Component [FE][medium]

Create `src/components/private-route.tsx` to protect routes that require authentication.
- This component will receive `children` as props.
- It will use the `useAuth` hook to check `isAuthenticated`.
- If `!isAuthenticated`, it will render a `<Navigate to="/login" replace />` component.
- If `isAuthenticated`, it will render the `children`.

### Task 4: Create AuthRoute Component [FE][medium]

Create `src/components/auth-route.tsx` to guard authentication-related routes.
- This component will receive `children` as props.
- It will use the `useAuth` hook to check `isAuthenticated`.
- If `isAuthenticated`, it will render a `<Navigate to="/home" replace />` component.
- If `!isAuthenticated`, it will render the `children`.

### Task 5: Create Home Page Placeholder [FE][simple]

Create `src/pages/home.tsx` as a simple functional component that renders "Home Page". This will serve as a placeholder for future UI.

### Task 6: Create Login Page Placeholder [FE][simple]

Create `src/pages/login.tsx` as a simple functional component that renders "Login Page" and includes a button to call `login()` from `useAuth`. This will serve as a placeholder for future UI.

### Task 7: Update App.tsx with Routing and Guards [FE][medium]

Modify `src/App.tsx` to integrate the `AuthProvider` and define the application routes.
- Wrap the entire application with `<AuthProvider>`.
- Use `BrowserRouter` for routing.
- Define routes:
    - `/login` route using `AuthRoute` to wrap the `LoginPage`.
    - `/home` route using `PrivateRoute` to wrap the `HomePage`.
    - A default route (`/`) that redirects to `/home` if authenticated, or `/login` if not.

## Acceptance Criteria (from SA-875)

1.  **No auth → navigate to `/home`**: Redirected to `/login`
2.  **Authenticated → navigate to `/login`**: Redirected to `/home`
3.  **No auth → navigate to `/login`**: Page renders (stays)
4.  **Authenticated → navigate to `/home`**: Page renders (stays)
