# SA-848: Test agent - Implementation Plan

This plan outlines the steps to implement the login form UI component as described in `app/docs/spec/SA-848.md`.

## Phase: Frontend Development

### Task 1: Create `components` directory [FE][simple]
Create the directory `app/fe/src/components` to house the new React component.

### Task 2: Create `LoginForm.tsx` file [FE][simple]
Create the file `app/fe/src/components/LoginForm.tsx` with the basic functional React component structure, including necessary imports for `React` and `useState`.

### Task 3: Implement `LoginForm.tsx` state and handlers [FE][medium]
Inside `app/fe/src/components/LoginForm.tsx`:
- Initialize `username` and `password` state variables using `useState`.
- Implement `handleUsernameChange` to update the `username` state.
- Implement `handlePasswordChange` to update the `password` state.
- Implement `handleSubmit` to prevent default form submission and display an alert with the current username and password.

### Task 4: Create `LoginForm.css` file [FE][simple]
Create the file `app/fe/src/components/LoginForm.css`.

### Task 5: Implement `LoginForm.css` styles [FE][medium]
Inside `app/fe/src/components/LoginForm.css`, add the CSS rules for:
- `.login-container` (centering, full height, background)
- `.login-form` (card-style, padding, shadow, max-width)
- `.form-group` (spacing, flex column)
- `label` (margin, font styles)
- `input` (padding, border, radius, full width)
- `button` (background, color, padding, border, radius, full width, cursor, hover)
- `h2` (margin, text align, color)

### Task 6: Implement `LoginForm.tsx` JSX structure [FE][medium]
Inside `app/fe/src/components/LoginForm.tsx`:
- Import `LoginForm.css`.
- Add the JSX structure for the login form, including the `div.login-container`, `form.login-form`, `h2`, `div.form-group` for username and password, `label`, `input`, and `button`.
- Bind the `value` and `onChange` props of the input fields to their respective state variables and handlers.
- Bind the `onSubmit` prop of the form to the `handleSubmit` function.

### Task 7: Integrate `LoginForm` into `App.tsx` [FE][simple]
Modify `app/fe/src/App.tsx`:
- Import the `LoginForm` component.
- Replace the existing `<h1>Hello World</h1>` with `<LoginForm />`.

## Acceptance Criteria Checklist

- [ ] New `components` directory exists in `src/`
- [ ] `LoginForm.tsx` component file exists and exports a React component
- [ ] `LoginForm.css` file exists and is imported in `LoginForm.tsx`
- [ ] Form has a "Login" heading (h2)
- [ ] Form has a "Username" label and text input field
- [ ] Form has a "Password" label and password input field (characters hidden)
- [ ] Form has a "Login" submit button
- [ ] Typing in username field updates the component state
- [ ] Typing in password field updates the component state
- [ ] Clicking submit button prevents page reload
- [ ] Clicking submit button displays an alert with format: `Username: [value], Password: [value]`
- [ ] Form is visually centered on the page
- [ ] Form has a card-style appearance (white background, shadow, border radius)
- [ ] Input fields and button are styled and easy to use
- [ ] `App.tsx` imports and renders the `LoginForm` component
