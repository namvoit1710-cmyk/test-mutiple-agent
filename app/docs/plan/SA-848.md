# SA-848: Test agent - Implementation Plan

## Goal

Create a simple login form UI component in the React frontend with username and password fields. When submitted, the form should display an alert showing the values entered by the user.

## Implementation Plan

### Task 1: Create `components` directory [FE][simple]
*   **Action:** Create the `app/fe/src/components` directory.

### Task 2: Create `LoginForm.tsx` file [FE][simple]
*   **Action:** Create the empty file `app/fe/src/components/LoginForm.tsx`.

### Task 3: Implement `LoginForm.tsx` basic structure and imports [FE][medium]
*   **Action:** Add the functional React component definition, import `useState` from React, and import `./LoginForm.css`.

### Task 4: Implement `LoginForm.tsx` state management [FE][medium]
*   **Action:** Add `useState` hooks for `username` (initial value: empty string) and `password` (initial value: empty string).

### Task 5: Implement `LoginForm.tsx` event handlers [FE][medium]
*   **Action:** Implement `handleUsernameChange` to update `username` state, `handlePasswordChange` to update `password` state, and `handleSubmit` to prevent default form submission and display an alert with the current username and password values.

### Task 6: Implement `LoginForm.tsx` JSX structure [FE][medium]
*   **Action:** Add the specified JSX structure for the login form, including a `div` with class `login-container`, a `form` with class `login-form`, an `h2` for the title, two `div`s with class `form-group` for username and password fields (each with a `label` and `input`), and a `button` for submission. Bind input `value` attributes to state variables and `onChange` to respective handlers. Bind `onSubmit` to `handleSubmit`.

### Task 7: Create `LoginForm.css` file [FE][simple]
*   **Action:** Create the empty file `app/fe/src/components/LoginForm.css`.

### Task 8: Implement `LoginForm.css` container and form styles [FE][medium]
*   **Action:** Add CSS rules for `.login-container` (flex display, center alignment, full viewport height, background color) and `.login-form` (white background, padding, border radius, box shadow, max-width, width).

### Task 9: Implement `LoginForm.css` form group, label, and input styles [FE][medium]
*   **Action:** Add CSS rules for `.form-group` (margin-bottom, flex column display), `label` (margin-bottom, font-weight, font-size), and `input` (padding, border, border-radius, font-size, full width).

### Task 10: Implement `LoginForm.css` button and heading styles [FE][medium]
*   **Action:** Add CSS rules for `button` (background, color, padding, border, border-radius, font-size, cursor, full width, margin-top, hover state) and `h2` (margin-bottom, text-align, color).

### Task 11: Update `App.tsx` to import `LoginForm` [FE][simple]
*   **Action:** Open `app/fe/src/App.tsx` and add the import statement for `LoginForm` at the top of the file.

### Task 12: Update `App.tsx` to render `LoginForm` [FE][simple]
*   **Action:** In `app/fe/src/App.tsx`, replace the existing `<h1>Hello World</h1>` with `<LoginForm />` within the main `App` component's return statement.

## Acceptance Criteria

*   New `components` directory exists in `src/`
*   `LoginForm.tsx` component file exists and exports a React component
*   `LoginForm.css` file exists and is imported in `LoginForm.tsx`
*   Form has a "Login" heading (h2)
*   Form has a "Username" label and text input field
*   Form has a "Password" label and password input field (characters hidden)
*   Form has a "Login" submit button
*   Typing in username field updates the component state
*   Typing in password field updates the component state
*   Clicking submit button prevents page reload
*   Clicking submit button displays an alert with format: `Username: [value], Password: [value]`
*   Form is visually centered on the page
*   Form has a card-style appearance (white background, shadow, border radius)
*   Input fields and button are styled and easy to use
*   `App.tsx` imports and renders the `LoginForm` component
