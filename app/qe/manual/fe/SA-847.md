## Module: LoginForm

File: `app/fe/src/components/LoginForm.tsx`

| TC | Description | Input | Expected Output |
|----|-------------|-------|-----------------|
| TC-001 | Verify LoginForm renders with a "Login" heading. | Render `<LoginForm />` | `<h2>Login</h2>` is visible. |
| TC-002 | Verify LoginForm renders with a "Username" label and text input. | Render `<LoginForm />` | `<label htmlFor="username">Username</label>` and `<input type="text" id="username" />` are visible. |
| TC-003 | Verify LoginForm renders with a "Password" label and password input. | Render `<LoginForm />` | `<label htmlFor="password">Password</label>` and `<input type="password" id="password" />` are visible, characters are hidden. |
| TC-004 | Verify LoginForm renders with a "Login" submit button. | Render `<LoginForm />` | `<button type="submit">Login</button>` is visible and enabled. |
| TC-005 | Verify handleUsernameChange updates the username state. | Type "testuser" into the username input. | Internal `username` state is "testuser". |
| TC-006 | Verify handlePasswordChange updates the password state. | Type "testpass" into the password input. | Internal `password` state is "testpass". |
| TC-007 | Verify handleSubmit prevents default form submission. | Click the "Login" button. | `event.preventDefault()` is called; no page reload occurs. |
| TC-008 | Verify handleSubmit displays an alert with correct username and password. | Username: "user1", Password: "pass1". Click "Login" button. | `alert("Username: user1, Password: pass1")` is called. |
| TC-009 | Verify handleSubmit displays an alert with empty username and password. | Username: "", Password: "". Click "Login" button. | `alert("Username: , Password: ")` is called. |
| TC-010 | Verify handleSubmit displays an alert with special characters in username and password. | Username: "!@#$", Password: "%^&*". Click "Login" button. | `alert("Username: !@#$, Password: %^&*")` is called. |
