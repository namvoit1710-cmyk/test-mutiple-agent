# Manual Test Cases: SA-875 [FE]

Source spec: `app/docs/spec/SA-875.md`

### TC-001: Unauthenticated user attempts to access /home

- **Priority:** Critical
- **Type:** Happy Case
- **Preconditions:** User is unauthenticated (no token in local storage).
- **Test Data:** None specific, relies on absence of token.

| # | Step | Expected Result |
|---|------|-----------------|
| 1 | Open the application in a new browser session or incognito window. | The application loads. |
| 2 | Manually navigate to the `/home` URL. | The application automatically redirects the user to the `/login` page. |

- **Postconditions:** User is on the `/login` page.

### TC-002: Authenticated user attempts to access /login

- **Priority:** Critical
- **Type:** Happy Case
- **Preconditions:** User is authenticated (a valid token exists in local storage).
- **Test Data:** A valid authentication token.

| # | Step | Expected Result |
|---|------|-----------------|
| 1 | Ensure a valid token is present in local storage. | Token is confirmed in local storage. |
| 2 | Manually navigate to the `/login` URL. | The application automatically redirects the user to the `/home` page. |

- **Postconditions:** User is on the `/home` page.

### TC-005: User logs out from /home

- **Priority:** High
- **Type:** Edge Case
- **Preconditions:** User is authenticated and on the `/home` page.
- **Test Data:** A valid authentication token.

| # | Step | Expected Result |
|---|------|-----------------|
| 1 | Ensure a valid token is present in local storage. | Token is confirmed in local storage. |
| 2 | Navigate to the `/home` page. | The user is on the `/home` page. |
| 3 | Perform a logout action (e.g., click a logout button, which should clear the token). | The token is cleared from local storage, and the user is redirected to the `/login` page. |

- **Postconditions:** User is on the `/login` page, and no token is present in local storage.

### TC-006: User logs in from /login

- **Priority:** High
- **Type:** Edge Case
- **Preconditions:** User is unauthenticated and on the `/login` page.
- **Test Data:** Valid login credentials (username/password).

| # | Step | Expected Result |
|---|------|-----------------|
| 1 | Ensure no token is present in local storage. | No token is present in local storage. |
| 2 | Navigate to the `/login` page. | The user is on the `/login` page. |
| 3 | Enter valid login credentials and submit the form. | A valid token is saved to local storage, and the user is redirected to the `/home` page. |

- **Postconditions:** User is on the `/home` page, and a valid token is present in local storage.
